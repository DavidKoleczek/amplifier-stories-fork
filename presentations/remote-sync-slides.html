Agent Discoveries

What The Agents Found

Each specialized agent uncovered critical insights that shaped the final design.

amplifier-expert
Session File Architecture
Discovered that transcript.jsonl is the resume source of truth, not events.jsonl. Only transcript needs immediate sync.
zen-architect
Block-Based Event Sourcing
Designed immutable SessionBlocks with sequence numbers and device IDs for multi-machine conflict detection.
zen-architect
Three-Layer Storage
HybridBlockStorage: always write to LocalBlockStorage first (fast), then background sync to CosmosBlockStorage. Never lose data.
modular-builder
Directory Exclusion System
Glob-based filtering so sensitive directories stay local-only. Pattern matching: ~/work/private/*

The Pivot

From CLI to Daemon

A critical architectural decision that changed everything.

Original Idea
Modify the CLI
Add sync during session
Changes affect everyone
→
Final Design
Standalone Rust daemon
Watches files externally
Zero CLI changes

"That means also we don't need to touch at all the CLI for the moment. We have the libraries... and it's less invasive. That means we're not breaking anybody. Full stop."

The Architecture

Final Design

A daemon-based approach with enterprise infrastructure.

Key Features

Architecture Highlights

Zero CLI Impact
Daemon runs separately. Users never see sync errors in their sessions.
Privacy First
Exclude folders, mark sessions local-only. Multi-tenant with opt-out.
W3C Trace Compatible
Full hierarchical tracing with OpenTelemetry Gen AI conventions.
Session Rehydration
Resume sessions from any machine. Your sessions are protected.
Offline-First
Write to disk first, sync in background. Never lose data.
Cross-Platform
Rust daemon for macOS, Windows, Linux. No runtime deps.
The Evolution

Design Through Iteration

Each phase built on the discoveries of the previous, guided by continuous AI exploration.

1

Requirements

amplifier-expert explained session architecture. Identified constraints: 2MB Cosmos limit, 100k+ token events.

2

Architecture

zen-architect produced 1,600+ lines of specs. Key: offline-first, sequence-based ordering.

3

Implementation

modular-builder created storage backends. Tests grew from 168 to 241.

4

Course Correction

Discovered tool_module approach was incorrect. This is a foundational library, not a tool.

5

Agent Access

SessionTool with safety guarantees. Never return full event data (100k+ tokens).

6

Integration

DirectoryExclusionFilter, HybridSessionStore, settings integration. Final: 1,873 insertions.

V1 Scope

Collect First, Restore Later
