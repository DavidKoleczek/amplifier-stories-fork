Step 1: Executor Replacement

Before: Dual Executors
// Two separate implementations
class PostgreSQLExecutor { ... }
class PGliteExecutor { ... }

// Environment detection
if (USE_PGLITE) {
  // PGlite path
} else {
  // PostgreSQL path
}
After: Single Executor
import Database from 'better-sqlite3';

class SQLiteExecutor {
  constructor(dbPath: string) {
    this.db = new Database(dbPath);
    this.db.pragma('journal_mode = WAL');
    this.db.pragma('foreign_keys = ON');
  }

  // Works everywhere
  query<T>(sql, params): T[] {
    return this.db.prepare(sql).all(params);
  }
}

Step 2: Migration File Conversion

42 migration files converted from PostgreSQL to SQLite SQL

Before: PostgreSQL
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  created_at TIMESTAMP 
    DEFAULT CURRENT_TIMESTAMP
);
After: SQLite
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  created_at TEXT 
    DEFAULT (datetime('now'))
);

Automated conversions: SERIAL → AUTOINCREMENT, TIMESTAMP → TEXT

Step 3: Query Parameterization

17 query files updated from PostgreSQL to SQLite parameter syntax

Before: $1, $2 Parameters
db.query(
  'SELECT * FROM users WHERE id = $1 AND role = $2',
  [userId, role]
)
After: ? Parameters
db.query(
  'SELECT * FROM users WHERE id = ? AND role = ?',
  [userId, role]
)

Regex-based search-and-replace + LSP code navigation for complex cases
